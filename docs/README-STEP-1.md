# Step 1 Summary

Step 1 is the approved baseline for Lentis.

This milestone establishes the initial design, project direction, and foundational source code. The purpose of this document is to make it clear that Step 1 is not a draft to keep editing casually. It is the protected starting point for all future work.

## Step 1 Objectives

Step 1 exists to:

- lock in the approved design direction
- preserve the current structure and information
- create a stable reference point for future development
- prevent accidental redesign before later project steps

## Current Rule

The Step 1 baseline should remain unchanged unless there is explicit approval to revise it.

Any work done before Step 2 and Step 3 should be additive and isolated.

## Allowed Additions Before Next Steps

- documentation
- repository setup
- branch strategy setup
- non-destructive UI polish
- planning for SEO, CI/CD, and deployment

## Not Allowed at This Stage

- redesigning the application
- changing approved content
- changing layout structure
- altering the visual identity already approved
- editing source files in ways that move beyond Step 1 scope

## Reference Documents

- `docs/step-1-baseline.md`
- `docs/branching-strategy.md`
