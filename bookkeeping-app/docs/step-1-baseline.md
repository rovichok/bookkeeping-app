# Step 1 Baseline

This document defines the approved Step 1 baseline for Lentis.

Its purpose is to preserve what has already been approved and reduce accidental drift as new work begins.

## Status

Step 1 is approved and frozen.

## What Step 1 Includes

The Step 1 baseline includes:

- the current approved design direction
- the current approved visual identity
- the current approved layout and structure
- the current approved information and messaging
- the current approved source code state

## Baseline Protection Rule

This baseline should not be changed directly.

All future work should happen in isolated branches and should be evaluated against this document before merging.

## Approved Principles

The current Step 1 version is intended to be:

- clean
- professional
- trustworthy
- modern
- restrained
- conversion-capable without being aggressive

## What Must Not Change Without Explicit Approval

The following are considered protected at this stage:

- core layout structure
- content hierarchy
- approved wording and information
- section ordering
- major spacing decisions
- visual identity direction
- overall design tone

## What May Be Added Before Step 2 and Step 3

Only non-destructive additions are allowed at this time:

- documentation
- GitHub/repository setup
- branch strategy
- milestone tagging
- subtle visual polish
- internal planning documents

## Examples of Allowed Visual Polish

Allowed:

- smoother hover transitions
- better focus states
- gentle fade/slide entrances
- refined button interaction
- subtle card shadow enhancement

Not allowed:

- redesigning page layouts
- changing content blocks
- moving sections around
- changing colors broadly
- replacing typography choices
- rewriting messaging
